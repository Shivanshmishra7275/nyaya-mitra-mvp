# ⚖️ Nyaya Mitra — AI-Powered Legal Assistant for Every Indian Citizen

> **"Nyaya Mitra" (न्याय मित्र) means "Friend of Justice."**
>
> A full-stack Legal Tech SaaS that makes India's new criminal justice laws — the **Bharatiya Nyaya Sanhita (BNS)**, **Bharatiya Nagarik Suraksha Sanhita (BNSS)**, and **Bharatiya Sakshya Adhiniyam (BSA)** — accessible to every ordinary citizen and advocate.

---

## 🌟 The Problem

India replaced the colonial-era IPC, CrPC, and Indian Evidence Act in 2023. Millions of citizens, lawyers, and police officers now navigate unfamiliar section numbers, new procedures, and restructured offences — with no easy guide. **Nyaya Mitra bridges that gap with AI.**

---

## ✨ Key Features

| Feature | Description |
|---|---|
| 🔍 **RAG-Powered Legal Q&A** | Ask any legal question in plain English. The backend performs semantic retrieval over **2,000+ chunked provisions** of BNS/BNSS/BSA stored in a local ChromaDB vector store, then generates a structured, citation-grounded answer via Google Gemini. |
| 📄 **AI Document Drafting** | Auto-generates five types of court-ready legal documents from user-provided facts: FIR Drafts, Bail Applications, Anticipatory Bail Petitions, Legal Notices, and Complaint Letters. |
| 🏛️ **BNS/BNSS Section Citations** | Every AI response includes the exact BNS/BNSS/BSA section numbers retrieved from the vector store — never hallucinated. |
| 📱 **Cross-Platform Frontend** | React Native + Expo frontend works on Web, Android, and iOS from a single codebase. |
| 🔒 **Local-First Vector DB** | ChromaDB runs entirely on your machine — no external vector database subscription needed. |
| ⚠️ **Built-in Legal Disclaimer** | Every response includes a mandatory professional disclaimer, ensuring responsible AI usage. |

## 🏗️ Tech Stack

### Backend
| Layer | Technology |
|---|---|
| **API Framework** | [FastAPI](https://fastapi.tiangolo.com/) — async Python, auto-generated OpenAPI docs |
| **AI Generation** | [Google Gemini 2.5 Flash](https://ai.google.dev/) via `google-generativeai` SDK |
| **Vector Database** | [ChromaDB](https://www.trychroma.com/) — local persistent store with HNSW cosine index |
| **Embedding Model** | [`all-MiniLM-L6-v2`](https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2) via `sentence-transformers` (384-dim, CPU-friendly) |
| **ETL Pipeline** | LangChain + PyPDF — chunks raw PDF law documents and ingests them into ChromaDB |
| **Data Validation** | Pydantic V2 — strict request/response schemas |
| **ASGI Server** | Uvicorn with hot-reload for development |
| **Local Database** | SQLite — session and query logging |

### Frontend
| Layer | Technology |
|---|---|
| **Framework** | [React Native](https://reactnative.dev/) with [Expo](https://expo.dev/) |
| **Navigation** | React Navigation |
| **Targets** | Web (browser), Android, iOS — single codebase |

## 🚀 Local Setup — Step-by-Step

Follow these instructions **exactly** to get Nyaya Mitra running on your machine from scratch.

### Prerequisites

- **Python 3.10+** — [Download](https://www.python.org/downloads/)
- **Node.js 18+ and npm** — [Download](https://nodejs.org/)
- **A Google Gemini API Key** — [Get one free at Google AI Studio](https://aistudio.google.com/app/apikey)
- **Git** — [Download](https://git-scm.com/)

---

### Step 1 — Clone the Repository

```bash
git clone https://github.com/your-username/nyaya-mitra.git
cd nyaya-mitra
```

---

### Step 2 — Create the Python Virtual Environment

It is strongly recommended to use a virtual environment to keep dependencies isolated.

```bash
# Create the virtual environment
python -m venv .venv

# Activate it (Windows PowerShell)
.\.venv\Scripts\Activate.ps1

# Activate it (Windows CMD)
.\.venv\Scripts\activate.bat

# Activate it (macOS / Linux)
source .venv/bin/activate
```

> ✅ Your terminal prompt should now show `(.venv)` indicating the environment is active.

---

### Step 3 — Install Python Dependencies

```bash
pip install -r requirements.txt
```

> ⏳ This will download FastAPI, ChromaDB, SentenceTransformers, Gemini SDK, and all other dependencies. The first install may take a few minutes as `sentence-transformers` downloads its model weights.

---

### Step 4 — Configure Your Environment Variables

Create a file named **`.env`** in the **project root** (same folder as `main.py`):

```bash
# Create the file
touch .env        # macOS / Linux
type nul > .env   # Windows CMD
```

Open `.env` in any text editor and add the following:

```env
# ── Required ────────────────────────────────────────────────────────────────
GEMINI_API_KEY=your_google_gemini_api_key_here

# ── Optional (these are the defaults — only change if needed) ────────────────
CHROMA_DB_PATH=./chroma_db
CHROMA_COLLECTION_NAME=nyaya_mitra_legal
EMBEDDING_MODEL=all-MiniLM-L6-v2
RAG_TOP_K=15
```

> ⚠️ **Never commit your `.env` file to Git.** It is already included in `.gitignore`.

---

### Step 5 — Build the Vector Database (Run Once)

This is a **one-time step**. The ETL pipeline reads the source PDF law documents from the `Raw_Data/` folder, splits them into 1,000-character chunks, embeds them using `all-MiniLM-L6-v2`, and persists the vectors to ChromaDB.

```bash
python etl_pipeline.py
```

> ✅ **Expected output:** The script will print progress logs as it ingests documents and finally confirm the number of chunks added to the `nyaya_mitra_legal` collection. The `chroma_db/` directory will be created automatically.
>
> ⏳ This may take **2–5 minutes** on first run depending on the number of PDF documents. You only need to run this **once** — or again after adding new source documents.

---

### Step 6 — Start the Backend Server

```bash
uvicorn main:app --reload --port 8000
```

> ✅ **Expected output:**
> ```
> INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
> INFO:     Application startup complete.
> ```
>
> 📖 **Interactive API Docs** (Swagger UI) are available at: **[http://localhost:8000/docs](http://localhost:8000/docs)**

---

### Step 7 — Start the Frontend (New Terminal)

Open a **new, separate terminal window** and run:

```bash
cd nyaya-mitra-app
npm install
npx expo start
```

Once the Expo development server is running, press **`w`** in the terminal to open the app in your **web browser**.

> ✅ **Expected output:** A QR code and a menu of options will appear. Press `w` for web, `a` for Android emulator, or `i` for iOS simulator.
>
> 📱 To run on a physical phone, install the **Expo Go** app ([Android](https://play.google.com/store/apps/details?id=host.exp.exponent) | [iOS](https://apps.apple.com/app/expo-go/id982107779)) and scan the QR code.

## 📡 API Reference

### `POST /api/v1/legal-query`
Performs semantic RAG retrieval over the Indian legal corpus and generates a structured Gemini response.

```json
// Request Body
{
  "user_query": "What is the punishment for theft under BNS?",
  "top_k": 15,
  "session_id": "optional-uuid-string"
}
```

### `POST /api/v1/draft-document`
Generates a court-ready legal document from user-provided facts.

```json
// Request Body
{
  "document_type": "bail_application",
  "facts": "The accused was arrested on 1st March 2026 in connection with...",
  "accused_name": "Ramesh Kumar",
  "court": "Sessions Court, Delhi",
  "sections_charged": ["BNS Section 303", "BNS Section 61"],
  "session_id": "optional-uuid-string"
}
```

**Supported `document_type` values:**
- `bail_application`
- `anticipatory_bail`
- `fir_draft`
- `legal_notice`
- `complaint_letter`
